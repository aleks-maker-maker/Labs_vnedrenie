## 📚 2. Теоретические сведения

### 🔹 Общие сведения о ГОСТ 34.602-89

**ГОСТ 34.602-89** — государственный стандарт, который распространяется на автоматизированные системы (АС) для автоматизации различных видов деятельности: управление, проектирование, исследование и т.п., включая их сочетания.

**Основное назначение стандарта** — установление состава, содержания и правил оформления документа «Техническое задание на создание (развитие или модернизацию) системы».

---

### 🔹 Значение технического задания

**Техническое задание (ТЗ) на АС** является основным документом, определяющим:

- ✅ Требования к создаваемой системе
- ✅ Порядок создания (развития или модернизации) автоматизированной системы
- ✅ В соответствии с ТЗ проводится разработка АС и её приемка при вводе в действие

---

### 🔹 Объект стандартизации

ГОСТ 34.602-89 регламентирует структуру ТЗ на создание именно **СИСТЕМЫ**, в которую входят:

| Компонент | Описание |
|-----------|----------|
| **Программное обеспечение (ПО)** | Прикладные и системные программы |
| **Аппаратное обеспечение** | Компьютеры, серверы, сетевое оборудование |
| **Персонал (люди)** | Специалисты, работающие с системой |
| **Автоматизируемые процессы** | Бизнес-процессы, подлежащие автоматизации |

---

## 📋 3. Структура технического задания согласно ГОСТ 34.602-89

Стандарт устанавливает, что техническое задание должно включать следующие разделы:

| № | Название раздела | Краткое описание |
|---|------------------|------------------|
| 1 | **Общие сведения** | Полное наименование системы, заказчик, разработчик, плановые сроки, основание для разработки |
| 2 | **Назначение и цели создания системы** | Описание функционального назначения системы, цели, для достижения которых она создаётся |
| 3 | **Характеристика объектов автоматизации** | Описание текущего состояния автоматизируемых процессов, существующие проблемы и недостатки |
| 4 | **Требования к системе** | Функциональные и нефункциональные требования, требования к надёжности, безопасности, интерфейсу |
| 5 | **Состав и содержание работ по созданию системы** | Этапы и сроки выполнения работ, перечень необходимых исследований и разработок |
| 6 | **Порядок контроля и приемки системы** | Виды испытаний, критерии приёмки, перечень документации, предоставляемой при сдаче |
| 7 | **Требования к подготовке объекта автоматизации** | Необходимые мероприятия для внедрения системы: обучение персонала, подготовка помещений |
| 8 | **Требования к документированию** | Перечень разрабатываемой документации, требования к её оформлению |
| 9 | **Источники разработки** | Перечень нормативных документов, методических материалов, использованных при составлении ТЗ |

---

## 🔍 4. Детальное описание разделов ТЗ

### 🔹 Раздел 1. Общие сведения

В данном разделе указываются:

| Пункт | Содержание |
|-------|------------|
| **Наименование системы** | Полное и краткое наименование автоматизированной системы |
| **Заказчик** | Организация или лицо, финансирующее разработку |
| **Разработчик** | Организация или лицо, выполняющее разработку |
| **Основание для разработки** | Нормативный документ, приказ или решение руководства |
| **Плановые сроки** | Начало и окончание работ по созданию системы |
| **Порядок финансирования** | Источники и порядок финансирования работ |

В данном разделе описываются:

| **Цели создания** | Каких результатов планируется достичь |
| **Критерии эффективности** | Показатели, по которым оценивается достижение целей |

---

### 🔹 Раздел 3. Характеристика объектов автоматизации

В данном разделе описываются:

|-------|------------|
| **Текущее состояние** | Описание существующих процессов и технологий |
| **Проблемы** | Недостатки и ограничения текущей системы |
| **Обоснование необходимости** | Почему требуется создание АС |

---

### 🔹 Раздел 4. Требования к системе

В данном разделе содержатся:

| Вид требований | Содержание |
|----------------|------------|
| **Функциональные** | Что должна делать система (функции, алгоритмы) |
| **Технические** | Требования к аппаратному обеспечению, пропускной способности |
| **Эргономические** | Требования к интерфейсу, удобству использования |
| **Безопасности** | Требования к защите информации, резервированию |
| **Надёжности** | Время восстановления, вероятность безотказной работы |

| **4. Опытная эксплуатация** | Пробная эксплуатация, доработка |
| **5. Ввод в действие** | Приёмка, ввод системы в промышленную эксплуатацию |

## 📋 5. Порядок разработки, согласования и утверждения ТЗ

### 🔹 Этап 1. Разработка ТЗ

| Действие | Содержание |
|----------|------------|
| **Формирование требований** | Заказчик определяет основные требования к системе |
| **Разработка проекта ТЗ** | Разработчик готовит первый вариант ТЗ |
| **Согласование с заказчиком** | Обсуждение, уточнение, внесение изменений |

### 🔹 Этап 2. Согласование ТЗ

| Действие | Содержание |
|----------|------------|
| **Внутреннее согласование** | Согласование внутри организации-разработчика |
| **Согласование с заказчиком** | Утверждение окончательного варианта |
| **Экспертиза** | При необходимости — независимая экспертиза |

### 🔹 Этап 3. Утверждение ТЗ

| Действие | Содержание |
|----------|------------|
| **Подписание ТЗ** | Руководители заказчика и разработчика |
| **Регистрация** | Присвоение номера, регистрация в системе документооборота |
| **Начало работ** | ТЗ является основанием для начала разработки |
