## 📚 2. Теоретические сведения

### 🔹 2.1. Что такое драйвер

**Драйвер** (от англ. *driver*) — это компьютерная программа, при помощи которой операционная система получает доступ к аппаратному обеспечению устройства и управляет им. Проще говоря, это связующее звено между компьютером и внешним устройством.

#### 🧠 Зачем нужен драйвер?

Без драйвера операционная система не знает:
- ❓ как общаться с устройством;
- ❓ какие команды ему отправлять;
- ❓ как интерпретировать данные, поступающие от устройства.

**📌 Примеры устройств, требующих драйверы:**
- 🎮 Видеокарты (NVIDIA, AMD, Intel)
- 🖨️ Принтеры и сканеры
- 📶 Сетевые адаптеры (Wi-Fi, Ethernet, Bluetooth)
- 🔊 Звуковые карты
- 🔌 Контроллеры USB
- 🖥️ Материнские платы (чипсет)
- 🖱️ Периферийные устройства (мыши, клавиатуры, веб-камеры)

---

### 🔹 2.2. Где хранятся драйверы в Windows

В операционной системе Windows драйверы физически хранятся в системной папке:
📁 C:\Windows\System32\drivers\
Дополнительные файлы, связанные с драйверами (информационные, библиотеки), могут находиться в:
📁 C:\Windows\System32
📁 C:\Windows\INF\

#### 📄 Расширения файлов драйверов

| Расширение | Описание |
|------------|----------|
| `.sys` | ⚙️ Основной исполняемый файл драйвера (системный) |
| `.drv` | ⚙️ Файл драйвера устройства (устаревший формат) |
| `.dll` | 📚 Библиотека, используемая драйвером для работы |
| `.vxd` | 🧬 Виртуальный драйвер (для Windows 9x) |
| `.inf` | 📋 Информационный файл установки — содержит инструкции для инсталляции драйвера |

---

### 🔹 2.3. Реестр Windows и драйверы

Информация об установленных драйверах хранится в системном реестре по следующим путям:
📂 HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services
📂 HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Class\

Каждый драйвер регистрируется в реестре со своим уникальным идентификатором, что позволяет системе управлять загрузкой драйверов при старте ОС.

## 🛠️ 3. Способы установки и обновления драйверов

Существует **два основных подхода** к установке драйверов:

| Подход | Описание |
|--------|----------|
| 🔍 **Поиск драйвера для конкретного устройства** | Точная идентификация устройства и загрузка драйвера с сайта производителя |
| 📦 **Установка пакета драйверов** | Массовая установка драйверов для всех устройств с помощью специализированных сборок или утилит |

---

### 🔍 3.1. Поиск драйвера для конкретного устройства

#### 🧰 Способ 1. С помощью утилит-анализаторов

Программы типа **AIDA64**, **CPU-Z**, **HWiNFO** автоматически определяют всё установленное оборудование и показывают его точное название, что упрощает поиск драйвера на сайте производителя.

**Пример работы:**
- Запускается AIDA64 → раздел «Устройства» → находится нужный компонент
- Точное название устройства копируется и вставляется в поиск на официальном сайте производителя

---

#### 🖥️ Способ 2. Через Диспетчер устройств Windows (стандартное средство)

**Путь к Диспетчеру устройств:**
- `Пуск` → `Панель управления` → `Диспетчер устройств`
- Или: `Win + X` → `Диспетчер устройств`

Устройства, для которых не установлен драйвер, отображаются с **жёлтым значком восклицания** (`⚠️`).

**Обновление драйвера через Диспетчер устройств:**
1. ПКМ по устройству → «Обновить драйвер»
2. Выбрать «Автоматический поиск драйверов» (Windows ищет в своей базе и через Центр обновления)
3. Или «Выполнить поиск драйверов на этом компьютере» (если драйвер уже скачан)

---

#### 🆔 Способ 3. Поиск по ID устройства (Hardware ID)

Каждое устройство имеет уникальный идентификатор — **Hardware ID**.

**Как его получить:**
1. Открыть Диспетчер устройств
2. ПКМ по устройству → «Свойства»
3. Перейти на вкладку «Сведения»
4. Выбрать из списка «ИД оборудования»
5. Скопировать строку вида: PCI\VEN_10DE&DEV_1C82&SUBSYS_00000000&REV_A1
где `VEN` — код производителя, `DEV` — код устройства.

6. Вставить скопированный ID в поиск на сайтах:
- `devid.info`
- `driveridentifier.com`
- `pcidatabase.com`

Этот способ особенно полезен для экзотических или старых устройств, для которых сложно найти драйвер по названию.

---

### 📦 3.2. Установка пакета драйверов

Пакет драйверов — это готовый набор для массовой установки, который позволяет не искать каждый драйвер отдельно. Особенно удобен при переустановке операционной системы.

#### Популярные сборки и утилиты

| Утилита | Описание |
|---------|----------|
| **DriverPack Solution** | Автоматически определяет устройства и устанавливает драйверы из своей базы. Существует в онлайн- и офлайн-версиях. |
| **Snappy Driver Installer** | Аналог DriverPack, работает офлайн, содержит большой архив драйверов. |
| **3DP Chip** | Лёгкая утилита, которая находит драйверы для сетевых устройств, звука и видео. |
| **Официальные утилиты производителей** | Dell Update, ASUS DriverHub, Lenovo System Update, HP Support Assistant |

---

### 🔧 3.3. Ручная установка драйвера

**Порядок действий:**
1. ⬇️ Скачать драйвер с **официального сайта производителя** (критично для безопасности!)
2. 🖱️ Если скачан `.exe` — запустить установщик и следовать инструкциям
3. 🖱️ Если скачан `.inf` — через Диспетчер устройств: ПКМ → «Обновить драйвер» → «Выполнить поиск на этом компьютере» → указать папку с файлами
4. 🔄 Перезагрузить компьютер (если требуется)

## ❓ 6. Контрольные вопросы

### 1️⃣ Что такое драйвер? Где он хранится в Windows?

**Драйвер** — это программа-посредник между операционной системой и аппаратным устройством. Он позволяет ОС управлять устройством и обмениваться с ним данными.

В Windows драйверы хранятся преимущественно в папке: C:\Windows\System32\drivers\
Информационные файлы `.inf` — в `C:\Windows\INF\`, а библиотеки `.dll` — в `C:\Windows\System32\`.

---

### 2️⃣ Какие способы обновления и установки драйверов существуют?

Основные способы:
1. 🔄 **Автоматический** — через Центр обновления Windows (Windows Update)
2. 🖥️ **Полуавтоматический** — через Диспетчер устройств с поиском в интернете
3. 🖱️ **Ручной** — скачивание драйвера с официального сайта производителя и установка вручную
4. 📦 **Пакетный** — с помощью специализированных утилит (DriverPack, Snappy Driver Installer)
5. 🆔 **По ID устройства** — поиск по Hardware ID на специализированных сайтах

---

### 3️⃣ Что такое Hardware ID и как его использовать?

**Hardware ID** — это уникальный идентификатор устройства в виде строки: PCI\VEN_XXXX&DEV_XXXX&SUBSYS_XXXXXXXX&REV_XX

Он содержит информацию о производителе (VEN) и самом устройстве (DEV).  
Используется для точного поиска драйвера, когда другие способы не дают результата.  
Для его получения нужно: Диспетчер устройств → Свойства устройства → Вкладка «Сведения» → ИД оборудования.  
Затем найденный ID вставляется в поиск на сайтах типа `devid.info`.
