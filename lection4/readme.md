## 📚 2. Теоретические сведения

Программы для диагностики компьютера позволяют проверить конфигурацию компьютера (количество памяти, её использование, типы дисков и т.д.), а также проверить работоспособность устройств компьютера (прежде всего жестких дисков). Они позволяют выявить «намечающиеся» дефекты дисков (возникающие из-за износа магнитной поверхности диска) и предотвратить потерю данных, хранящихся на диске.

### 🔹 Программа PC Wizard

PC Wizard — программа, которая предоставляет обширную информацию обо всех установленных на компьютере компонентах: память, материнская плата, устройства для хранения и записи информации, видеоподсистема, сетевые устройства, модемы, принтеры и т.д., включая разнообразные данные об операционной системе — версию системы, установленные шрифты, библиотеки, WinSock, активные процессы, имеющиеся модули и сервисы и т.п.

Кроме этого, PC Wizard позволяет протестировать быстродействие процессора, жестких дисков и CD/DVD-привода. Поддерживаются 32- и 64-битные ОС.

### 🔹 Средство диагностики DirectX (dxdiag)

Программа отображает сведения о компонентах и драйверах интерфейса Microsoft DirectX. Позволяет проверить работу аудио- и видеокарты, а также установить связь с мультимедийными службами. С помощью средства диагностики также можно отключить некоторые средства аппаратного ускорения.

## ❓ 6. Ответы на контрольные вопросы

### 1. Причины возникновения проблем с совместимостью ПО

Проблемы с совместимостью программного обеспечения могут возникать по следующим причинам:

- **Несовместимость версий:** программа разработана для одной версии ОС, а запускается на другой (например, приложение для Windows 7 пытается работать на Windows 10/11).
- **Отсутствие необходимых библиотек:** программа требует наличие определённых библиотек (например, Microsoft Visual C++ Redistributable, .NET Framework), которые не установлены в системе.
- **Разрядность ОС:** 32-битное приложение может работать на 64-битной системе, но не наоборот.
- **Конфликт драйверов:** несовместимость драйверов с аппаратным обеспечением или другими драйверами.
- **Устаревшее ПО:** программа не обновлялась и не поддерживает новые версии ОС или новые аппаратные компоненты.
- **Аппаратные требования:** программа требует наличия определённого оборудования (например, дискретной видеокарты, большого объёма ОЗУ), которого нет на данном ПК.

### 2. Методы уменьшения проблем с совместимостью ПО

Для уменьшения проблем с совместимостью программного обеспечения используются следующие методы:

- **Запуск в режиме совместимости:** в Windows можно запустить программу в режиме совместимости с более старой версией ОС (правой кнопкой → «Свойства» → «Совместимость»).
- **Установка необходимых библиотек:** установка Microsoft Visual C++ Redistributable, .NET Framework, DirectX и других компонентов.
- **Использование виртуальных машин:** запуск устаревших программ в виртуальной машине с нужной версией ОС.
- **Обновление программы:** установка последней версии ПО, которая поддерживает текущую ОС.
- **Обновление драйверов:** установка актуальных версий драйверов для всех устройств.
- **Использование совместимых аналогов:** замена программы на альтернативу, которая работает на текущей ОС.
- **Настройка параметров совместимости в Windows:** использование встроенных средств устранения неполадок совместимости.
