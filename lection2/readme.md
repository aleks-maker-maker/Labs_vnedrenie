# 📘 Как создать папку в репозитории GitHub (веб-интерфейс)

## Главный секрет

> **В GitHub нельзя создать пустую папку. Папка создаётся автоматически, когда вы создаёте файл внутри неё.**

---

## 🔧 Пошаговая инструкция

### Шаг 1. Нажмите "Add file"

В корне репозитория нажмите зелёную кнопку **"Add file"** → выберите **"Create new file"**

---

### Шаг 2. Напишите путь к файлу

В поле **"Name your file"** напишите:

```
НАЗВАНИЕ_ПАПКИ/НАЗВАНИЕ_ФАЙЛА.расширение
```

**Пример:**
```
Lection2/Lab2.docx
```

После этого GitHub **сам создаст папку** `Lection2` и положит в неё файл `Lab2.docx`.

---

### Шаг 3. Сохраните изменения

Внизу страницы напишите комментарий и нажмите **"Commit new file"**

---

### Шаг 4. Чтобы создать вложенную папку (папку внутри папки)

Напишите путь через `/`:

```
Lection2/img/.gitkeep
```

GitHub создаст:
- Папку `Lection2`
- Внутри неё папку `img`
- Внутри `img` файл `.gitkeep`

> Файл `.gitkeep` — это просто заглушка, чтобы GitHub отобразил пустую папку.

---

## 📸 Как это выглядит на экране

1. Вы в репозитории
2. Нажимаете **"Add file"** → **"Create new file"**
3. Появляется страница с текстовым полем
4. В верхнем поле (где написано `Name your file`) вы печатаете:

```
Lection2/Lab2.docx
```

5. Внизу нажимаете **"Commit new file"**

**Всё!** Папка и файл созданы.

---

## ❓ Почему нельзя создать папку отдельно?

В GitHub **нет кнопки "Create folder"**. Папки — это просто часть пути к файлу. Если файл лежит по пути `Lection2/Lab2.docx`, значит, папка `Lection2` существует.

---

## 📌 Главное правило

| Что написать в поле имени файла | Что создастся |
|--------------------------------|---------------|
| `Lection2/Lab2.docx` | Папка `Lection2`, внутри файл `Lab2.docx` |
| `Lection2/img/.gitkeep` | Папка `Lection2` → внутри папка `img` → внутри файл `.gitkeep` |
| `Lection3/Lab3.docx` | Папка `Lection3`, внутри файл `Lab3.docx` |
| `README.md` | Просто файл в корне (без папок) |
