## 📚 2. Теоретические сведения

### 🔹 Что такое групповые политики

**Групповые политики** — это механизм централизованного управления настройками операционной системы и приложений, который позволяет администраторам задавать параметры для групп пользователей и компьютеров. В локальной версии (локальные групповые политики) они применяются только к текущему компьютеру.

#### Основные категории политик:

| Категория | Описание |
|-----------|----------|
| **Конфигурация компьютера** | Политики, применяемые ко всем пользователям системы. Выполняются до входа пользователя в систему. |
| **Конфигурация пользователя** | Политики, применяемые только к текущему пользователю. Выполняются после входа пользователя в систему. |

---

### 🔹 Запуск редактора локальной групповой политики

Для запуска редактора локальной групповой политики необходимо:

1. Нажать кнопку **Пуск**.
2. В строке поиска ввести **gpedit.msc**.
3. Нажать клавишу Enter.

Откроется окно **Редактор локальной групповой политики**, разделённое на две части:
- **Левая панель** — древовидная структура категорий политик.
- **Правая панель** — политики выбранной категории с их текущим состоянием.

---

### 🔹 Настройка политики

Для настройки политики выполняется двойной щелчок по ней. Политика может принимать одно из следующих состояний:

| Состояние | Описание |
|-----------|----------|
| **Не задано** | Политика не действует (значение по умолчанию) |
| **Включить** | Политика активна, её настройки применяются |
| **Отключить** | Политика активна, но параметры отключены |

---

### 🔹 Фильтрация политик

Редактор локальной групповой политики содержит около 3000 политик. Для упрощения поиска используются следующие функции:

1. Ветви **Все параметры** в разделах «Конфигурация компьютера» и «Конфигурация пользователя» — отображают полный список политик.
2. **Фильтрация** — позволяет сузить список политик по ключевым словам.

#### Настройка фильтра:

1. Выбрать ветвь **Все параметры**.
2. В меню **Действие** выбрать **Параметры фильтра**.
3. Установить флажок **Включить фильтры по ключевым словам**.
4. Ввести ключевое слово (например, «мест»).
5. Выбрать уровень соответствия: **Любой**, **Все** или **Точный**.
6. Выбрать область поиска: **Заголовок параметра политики**, **Текст справки** или **Комментарий**.
7. Нажать кнопку **OK**.

---

### 🔹 Часть 2. Настройка окна безопасности Windows

Для настройки кнопок, отображаемых в окне безопасности Windows (вызываемом сочетанием Ctrl+Alt+Delete), были выполнены следующие действия:

1. Открыто окно «Редактор локальной групповой политики».
2. Последовательно раскрыты ветви: **Конфигурация пользователя** → **Административные шаблоны** → **Система** → **Варианты действий после нажатия CTRL+ALT+DEL**.
3. Для каждой из политик выполнено двойное нажатие и установлен переключатель «Включить»:

| Политика | Назначение |
|----------|------------|
| **Запретить изменение пароля** | Отключает кнопку «Сменить пароль» |
| **Запретить блокировку компьютера** | Отключает кнопку «Блокировать компьютер» |
| **Удалить диспетчер задач** | Отключает кнопку «Запустить диспетчер задач» |
| **Запретить завершение сеанса** | Отключает кнопку «Выйти из системы» |

После применения политик в окне безопасности Windows отобразились только разрешённые кнопки.

---

### 🔹 Часть 3. Настройка панели мест

Для настройки панели мест в диалоговых окнах «Сохранить как» и «Открыть» были выполнены следующие действия:

1. Открыто окно «Редактор локальной групповой политики».
2. Последовательно раскрыты ветви: **Конфигурация пользователя** → **Административные шаблоны** → **Компоненты Windows** → **Проводник Windows** → **Общее диалоговое окно открытия файлов**.
3. Дважды выполнено нажатие на политике **«Элементы, отображаемые в панели мест»**.
4. Установлен переключатель **«Включить»**.
5. В текстовые поля **Элемент 1 — Элемент 5** введены пути к папкам:
   - `C:\Users\Admin\Documents\Проекты`
   - `D:\Рабочие\Документы`
   - `\\server\shared\Общие`
   - `C:\Users\Admin\Desktop`
   - `E:\Архив`

После применения политики в диалоговых окнах «Сохранить как» и «Открыть» в панели мест появились значки указанных папок.

---

### 🔹 Часть 4. Увеличение размера списка недавних документов

Для изменения количества отображаемых недавних документов были выполнены следующие действия:

1. Открыто окно «Редактор локальной групповой политики».
2. Последовательно раскрыты ветви: **Конфигурация пользователя** → **Административные шаблоны** → **Компоненты Windows** → **Проводник Windows**.
3. Дважды выполнено нажатие на политике **«Максимальная длина списка "Недавние документы"»**.
4. Установлен переключатель **«Включить»**.
5. В счетчике **«Максимальная длина списка "Недавние документы"»** установлено значение **30** (по умолчанию 15).
6. Нажата кнопка **OK**.

После применения политики в меню «Пуск» → «Недавние документы» стали отображаться до 30 документов.

---

### 🔹 Часть 5. Включение диалога слежения за завершением работы

Для включения диалога слежения за завершением работы были выполнены следующие действия:

1. Открыто окно «Редактор локальной групповой политики».
2. Последовательно раскрыты ветви: **Конфигурация компьютера** → **Административные шаблоны** → **Система**.
3. Дважды выполнено нажатие на политике **«Отображать диалог слежения за завершением работы»**.
4. Установлен переключатель **«Включить»**.
5. В списке **«Диалог слежения за завершением работы должен отображаться»** выбрано значение **«Всегда»**.
6. Нажата кнопка **OK**.

После применения политики при выборе команды «Завершение работы» стало отображаться диалоговое окно с возможностью выбора причины завершения работы и добавления комментария.

## ❓ 5. Контрольные вопросы

### 1. Что такое групповые политики?

**Групповые политики** — это механизм управления настройками операционной системы и приложений, позволяющий администраторам задавать и применять параметры для компьютеров и пользователей. Они могут использоваться как на локальном компьютере (локальные групповые политики), так и в доменных сетях (через Active Directory).

---

### 2. Категории объектов групповых политик?

Основные категории групповых политик:

| Категория | Описание |
|-----------|----------|
| **Конфигурация компьютера** | Политики, применяемые ко всем пользователям системы. Включают настройки системы, безопасности, сети и т.д. |
| **Конфигурация пользователя** | Политики, применяемые к текущему пользователю. Включают настройки рабочего стола, меню «Пуск», программ и т.д. |

Каждая категория содержит разделы: **Программное обеспечение**, **Параметры Windows**, **Административные шаблоны**.

---

### 3. Разделы групповых политик

| Раздел | Описание |
|--------|----------|
| **Программное обеспечение** | Управление установкой и настройкой приложений |
| **Параметры Windows** | Настройки безопасности, сценарии входа/выхода, параметры сети |
| **Административные шаблоны** | Основной раздел с параметрами реестра, настройками Windows и компонентов |

---

### 4. Запуск групповых политик

Способы запуска редактора локальной групповой политики:

| Способ | Действие |
|--------|----------|
| **Командная строка** | Ввести `gpedit.msc` и нажать Enter |
| **Окно «Выполнить»** | Нажать Win+R → ввести `gpedit.msc` → OK |
| **Поиск в меню Пуск** | Нажать Пуск → ввести `gpedit.msc` → Enter |

> **Примечание:** Редактор локальной групповой политики доступен только в версиях Windows Pro, Enterprise и Education. В Windows Home он отсутствует.
